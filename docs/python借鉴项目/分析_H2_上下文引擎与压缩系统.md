## 一、核心发现摘要

### 1.1 与 claude-code / openclaw 的对比

| 维度 | claude-code | openclaw | hermes-agent |
|------|-------------|----------|--------------|
| **引擎抽象** | 内置三级递进 | ContextEngine ABC（接口） | **ContextEngine ABC（接口 + 生命周期钩子）** |
| **压缩算法** | 迭代摘要 + Token 预算 | 插件决定 | **5 阶段管线：剪枝→边界→摘要→组装→修复** |
| **Token 估算** | 精确（tiktoken） | 依赖 Provider | **粗估 `(chars+3)//4` + 图像 1600 token/张** |
| **摘要模型** | 主模型自身 | 可配置 | **辅助 LLM（可独立配置）+ 主模型回退** |
| **防注入** | 无显式机制 | 质量守卫 | **SUMMARY_PREFIX 防指令化 + redact_sensitive_text** |
| **提示词安全** | 无显式扫描 | 无显式扫描 | **10 条正则 + 10 个不可见 Unicode 检测** |
| **Prompt 缓存** | 无内置 | 无内置 | **Anthropic system_and_3 策略（4 断点）** |
| **工具输出剪枝** | 无 | 无 | **3 遍扫描：去重→摘要→截断参数** |
| **反抖动** | 无 | 无 | **连续 2 次 <10% 节省→暂停压缩** |
| **平台适配** | 单一 CLI | 单一 Web | **17 种平台提示词 + 环境感知** |

### 1.2 核心设计特点

1. **5 阶段压缩管线**：剪枝→边界→摘要→组装→修复，每阶段职责清晰
2. **双层摘要预算**：`_SUMMARY_RATIO=0.20`（按内容比例）+ `_SUMMARY_TOKENS_CEILING=12,000`（绝对上限）
3. **工具输出预剪枝**：3 遍扫描（去重→摘要→截断参数），无需 LLM 调用即可大幅缩减上下文
4. **结构化摘要模板**：13 个字段（Active Task、Goal、Completed Actions、Active State 等），支持迭代更新
5. **提示词安全扫描**：10 条正则 + 10 个不可见 Unicode 检测，阻止注入的上下文文件

## 二、模块架构分析

### 2.1 ContextEngine ABC 接口设计

```
ContextEngine (ABC)
├── 身份
│   └── name: str (abstract property)
│
├── Token 状态（run_agent.py 直接读取）
│   ├── last_prompt_tokens: int = 0
│   ├── last_completion_tokens: int = 0
│   ├── last_total_tokens: int = 0
│   ├── threshold_tokens: int = 0
│   ├── context_length: int = 0
│   └── compression_count: int = 0
│
├── 压缩参数（run_agent.py 预检查使用）
│   ├── threshold_percent: float = 0.75
│   ├── protect_first_n: int = 3
│   └── protect_last_n: int = 6
│
├── 核心接口（abstract）
│   ├── update_from_response(usage) → None
│   ├── should_compress(prompt_tokens?) → bool

│
├── 可选：预检查
│   ├── should_compress_preflight(messages) → bool  [default: False]
│   └── has_content_to_compress(messages) → bool    [default: True]
│
├── 可选：会话生命周期
│   ├── on_session_start(session_id, **kwargs) → None
│   ├── on_session_end(session_id, messages) → None

│
```

**设计亮点**：
- **属性直接暴露**：`last_prompt_tokens` 等作为类属性而非方法，run_agent.py 直接读取，零开销
- **可插拔工具**：`get_tool_schemas()` + `handle_tool_call()` 允许引擎向 Agent 暴露专用工具（如 LCM 的 `lcm_grep`）
- **配置驱动选择**：`context.engine` in config.yaml，默认 `"compressor"`

### 2.2 ContextCompressor 压缩算法

```
compress(messages, current_tokens, focus_topic)
    │
    ├── Phase 1: 工具输出预剪枝（无 LLM 调用）
    │   ├── Pass 1: 去重（MD5 哈希，保留最新完整副本）
    │   ├── Pass 2: 摘要替换（>200 字符的工具结果→1 行描述）
    │   └── Pass 3: 参数截断（>500 字符的 tool_call.arguments→JSON 安全截断）
    │
    ├── Phase 2: 边界确定
    │   ├── compress_start = protect_first_n（默认 3）
    │   │   └── _align_boundary_forward()：跳过孤立 tool 结果
    │   └── compress_end = _find_tail_cut_by_tokens()
    │       ├── 从末尾向前累积 token
    │       ├── soft_ceiling = budget × 1.5（避免切割大消息）
    │       ├── _align_boundary_backward()：不拆分 tool_call/result 组
    │       └── _ensure_last_user_message_in_tail()：确保最新用户消息在尾部
    │
    ├── Phase 3: 结构化摘要生成
    │   ├── _compute_summary_budget()：max(2000, min(content×0.20, ceiling))
    │   ├── _serialize_for_summary()：带 redact 的序列化
    │   ├── _generate_summary()：辅助 LLM 调用
    │   │   ├── 首次压缩：从零生成
    │   │   ├── 迭代更新：基于 _previous_summary 增量更新
    │   │   ├── 焦点引导：focus_topic → 60-70% 预算给焦点
    │   │   ├── 辅助模型失败 → 主模型回退（仅 1 次）
    │   │   └── 全部失败 → 静态回退占位符
    │   └── redact_sensitive_text()：输出二次脱敏
    │
    ├── Phase 4: 组装压缩消息列表
    │   ├── 头部消息（系统提示词 + 压缩注释）
    │   ├── 摘要消息（角色选择避免连续同角色）
    │   │   ├── 独立消息：选择 user/assistant 角色
    │   │   └── 合并到尾部：当两种角色都会冲突时
    │   └── 尾部消息（受保护区域）
    │
    └── Phase 5: 修复
        └── _sanitize_tool_pairs()：修复孤立 tool_call/result
            ├── 删除无对应 tool_call 的 tool result
            └── 为无对应 tool result 的 tool_call 插入桩结果
```

## 三、关键设计模式

### 3.1 五阶段压缩管线

**模式名称**：管道模式（Pipeline）

**应用位置**：`ContextCompressor.compress()` (行 1278-1479)

**设计亮点**：
- Phase 1（剪枝）是纯计算，无 LLM 调用，成本极低
- Phase 5（修复）确保 API 永远不会收到不匹配的 tool_call/result ID
- 每个 Phase 独立可测试

### 3.2 迭代摘要更新

**模式名称**：增量计算 + 快照链

**应用位置**：`_generate_summary()` (行 727-994)

**代码示例**：
```python
if self._previous_summary:
    # 迭代更新：保留现有信息，添加新进展
    prompt = f"""...
    PREVIOUS SUMMARY:
    {self._previous_summary}

    NEW TURNS TO INCORPORATE:
    {content_to_summarize}

    Update the summary using this exact structure. PRESERVE all existing
    information that is still relevant. ADD new completed actions...
    CRITICAL: Update "## Active Task" to reflect the user's most recent
    unfulfilled request — this is the most important field for task continuity.
    ..."""
else:
    # 首次压缩：从零生成
    prompt = f"""...
    TURNS TO SUMMARIZE:
    {content_to_summarize}
    ..."""
```

**设计亮点**：
- `_previous_summary` 在多次压缩间持久化，实现增量更新
- 摘要从压缩窗口中提取旧摘要体，避免重复摘要
- `Active Task` 字段被标记为"最重要的字段"，确保任务连续性

### 3.3 工具输出预剪枝（3 遍扫描）

**模式名称**：多遍扫描优化

**应用位置**：`_prune_old_tool_results()` (行 494-646)

**Pass 1 — 去重**：
```python
# MD5 哈希去重：同一文件读取 5 次只保留最新完整副本
content_hashes: dict = {}
for i in range(len(result) - 1, -1, -1):
    h = hashlib.md5(content.encode("utf-8", errors="replace")).hexdigest()[:12]
    if h in content_hashes:
        result[i] = {**msg, "content": "[Duplicate tool output — same content as a more recent call]"}
```

**Pass 2 — 摘要替换**：
```python
# 生成信息性 1 行摘要替代大块工具输出
summary = _summarize_tool_result(tool_name, tool_args, content)
# 例: [terminal] ran `npm test` -> exit 0, 47 lines output
# 例: [read_file] read config.py from line 1 (3,400 chars)
```

**Pass 3 — 参数截断**：
```python
# JSON 安全截断：解析→缩小长字符串→重新序列化
# 避免 MiniMax 等 Provider 因非法 JSON 返回 400
new_args = _truncate_tool_call_args_json(args)
```

### 3.4 提示词安全扫描

**模式名称**：正则规则引擎 + 不可见字符检测

**应用位置**：`_scan_context_content()` (行 55-73)

**威胁模式清单**：
```python
_CONTEXT_THREAT_PATTERNS = [
    (r'ignore\s+(previous|all|above|prior)\s+instructions', "prompt_injection"),
    (r'do\s+not\s+tell\s+the\s+user', "deception_hide"),
    (r'system\s+prompt\s+override', "sys_prompt_override"),
    (r'disregard\s+(your|all|any)\s+(instructions|rules|guidelines)', "disregard_rules"),
    (r'act\s+as\s+(if|though)\s+you\s+(have\s+no|don\'t\s+have)\s+(restrictions|limits|rules)', "bypass_restrictions"),
    (r'<!--[^>]*(?:ignore|override|system|secret|hidden)[^>]*-->', "html_comment_injection"),
    (r'<\s*div\s+style\s*=\s*["\'][\s\S]*?display\s*:\s*none', "hidden_div"),
    (r'translate\s+.*\s+into\s+.*\s+and\s+(execute|run|eval)', "translate_execute"),
    (r'curl\s+[^\n]*\$\{?\w*(KEY|TOKEN|SECRET|PASSWORD|CREDENTIAL|API)', "exfil_curl"),
    (r'cat\s+[^\n]*(\.env|credentials|\.netrc|\.pgpass)', "read_secrets"),
]

_CONTEXT_INVISIBLE_CHARS = {
    '\u200b', '\u200c', '\u200d', '\u2060', '\ufeff',  # 零宽字符
    '\u202a', '\u202b', '\u202c', '\u202d', '\u202e',  # 双向控制字符
}
```

**处理方式**：发现威胁时返回阻止消息而非原始内容：
```python
return f"[BLOCKED: {filename} contained potential prompt injection ({', '.join(findings)}). Content not loaded.]"
```

### 3.5 Anthropic Prompt 缓存策略

**模式名称**：断点注入 + 滚动窗口

**应用位置**：`apply_anthropic_cache_control()` (行 41-72)

**策略**：`system_and_3` — 最多 4 个缓存断点
```
断点 1: 系统提示词（跨轮次稳定）
断点 2-4: 最后 3 条非系统消息（滚动窗口）
```

**关键设计**：
- 纯函数，无类状态，无 AIAgent 依赖
- `deepcopy` 避免修改原始消息
- 支持 TTL 配置：`"5m"`（默认）或 `"1h"`
- 兼容原生 Anthropic 格式和 OpenAI 格式

### 3.6 技能索引两层缓存

**模式名称**：LRU + 磁盘快照

**应用位置**：`build_skills_system_prompt()` (行 718-949)

```
请求技能索引
    │
    ├── Layer 1: 进程内 LRU 缓存（OrderedDict, max=8）
    │   ├── 命中 → 直接返回
    │   └── 未命中 ↓
    │
    ├── Layer 2: 磁盘快照（.skills_prompt_snapshot.json）
    │   ├── manifest 匹配（mtime+size）→ 使用预解析元数据
    │   └── manifest 不匹配 ↓
    │
    └── Cold Path: 全文件系统扫描
        ├── 扫描所有 SKILL.md + DESCRIPTION.md
        ├── 条件过滤（platforms, requires, fallback_for）
        └── 写入磁盘快照 + 存入 LRU 缓存
```

```typescript
async function compress(messages: Message[]): Promise<Message[]> {
```

### 7.2 JSON 安全截断

**位置**：context_compressor.py 行 153-196

```python
def _truncate_tool_call_args_json(args: str, head_chars: int = 200) -> str:
    """Shrink long string values inside a tool-call arguments JSON blob
    while preserving JSON validity."""
    try:
        parsed = json.loads(args)
    except (ValueError, TypeError):
        return args  # 非 JSON 原样返回

    def _shrink(obj):
        if isinstance(obj, str):
            return obj[:head_chars] + "...[truncated]" if len(obj) > head_chars else obj
        if isinstance(obj, dict):
            return {k: _shrink(v) for k, v in obj.items()}
        if isinstance(obj, list):
            return [_shrink(v) for v in obj]
        return obj

    return json.dumps(_shrink(parsed), ensure_ascii=False)
```

**价值**：解决 MiniMax 等 Provider 因非法 JSON 返回 400 导致会话卡死的问题（#11762）

## 八、总结

### 8.1 核心收获

1. **ContextEngine ABC 是最完善的上下文引擎接口**：比 openclaw 的 ContextEngine 多出工具暴露、预检查、模型切换、状态展示等能力
2. **五阶段压缩管线是生产级实现**：工具输出预剪枝（无 LLM 调用）→ 边界确定 → 摘要生成 → 组装 → 修复，每阶段独立可测试
3. **SUMMARY_PREFIX 防指令化是独特设计**：通过精心措辞防止模型将历史请求误认为当前指令
4. **提示词安全扫描填补了空白**：10 条正则 + 10 个不可见 Unicode 检测，是 claude-code 和 openclaw 都没有的安全层
5. **17 种平台提示词展示了多平台适配的复杂度**：每种平台有独特的格式限制和文件发送能力

### 8.2 对 EvoAgent 的建议

1. **采纳 ContextEngine ABC**：定义完整的 TypeScript 接口，包含工具暴露和模型切换钩子
2. **采纳五阶段压缩管线**：特别是工具输出预剪枝（无 LLM 调用）和 tool pair 修复
3. **采纳结构化摘要模板**：13 字段模板 + 迭代更新 + 焦点引导
4. **采纳提示词安全扫描**：直接移植 10 条正则规则
5. **采纳 SUMMARY_PREFIX 防指令化**：防止压缩后模型重复已完成工作
6. **改进 Token 估算**：使用 `js-tiktoken` 替代 `(chars+3)//4`
